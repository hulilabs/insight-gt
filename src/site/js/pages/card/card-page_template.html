There was an interesting issue with the previous example regarding the dropdown component and how the card component handles the closing event. What happened was that when there was a dropdown menu contained inside a card and one of its options was clicked, the result was that the card component was closed and lost its focus.
This issue was related with how blur events work, because: in blur events, the relatedTarget corresponds to the element that GAINED focus. If you combine this with an element having visibility : hidden; that element won't be able to really "gain" focus, therefore, it won't appear as the card component blur event's relatedTarget. You can see more about this issue here.
